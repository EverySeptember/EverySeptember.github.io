# 三层架构

- 数据访问层（DAO）：直接与数据库进行交互
- 业务逻辑层（Service）：系统的主要业务逻辑写在这
- 视图层（View）：用于接收用户提交请求的代码

# MVC

MVC是对三层架构中视图层的应用

- **M**odel：模型
  - 业务模型：业务模型
  - 数据模型：实体类

- **V**iew：视图，为用户提供的界面，直接与用户交互
- **C**ontroller：用于将请求转发给相应的模型进行处理，并根据Model的计算结果给用户响应

# 应用

```markdown
.
├── dao：数据访问层-业务模型
│   └── impl：数据访问层-业务模型
├── entity：数据访问层-实体类
├── service：业务逻辑层
└── web：视图层
    ├── controller：servlet
    └── filter：过滤器
```
